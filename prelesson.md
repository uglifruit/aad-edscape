## The platforms you will need to download and sign up to

Before the day it would be great if you can come prepared with the following downloaded and/or signed up to:

**VS Code:** this will be our integrated development environment (IDE) - where we store our documents and code, and where we ‘invite’ in the AI to do the coding. https://code.visualstudio.com/. If you’re a die hard Cursor users that’s fine, but I’ll be using VS Code. 

**Github**: this is where we will be storing our code online to enable deployment. https://github.com/

**Vercel**: our deployment platform. Ensure you sign up to Github first, and then use your Github account to sign up to Vercel. https://vercel.com/

**Claude**. We will be using the Claude ecosystem throughout the day, so ensure you have an account. I would strongly recommend signing up to the Pro plan for the day (even if you don’t plan to continue beyond) as this enables you to have access to Claude Code desktop AND easily add it to VS Code. https://claude.ai

**Claude Desktop**: a great way to use Claude with simple access to Claude Code for desktop. https://support.claude.com/en/articles/10065433-installing-claude-desktop

**Claude Code**: our coding partner. We will actually be using some different AI models for this day (via Alibaba), but it will all run through Claude Code. This is to ensure we all have plenty of AI for the day and no one runs out. However, you do need to have downloaded and activated Claude Code in advance inside the Terminal. Ensure you have signed up to at least the Pro version so that you have access to Claude Code in the desktop. Follow the instructions on this page that relate to your operating system - Mac or Windows: https://code.claude.com/docs/en/overview. You can watch the video to show you how.

**Node**: our package manager. Don’t worry too much if this doesn’t make sense, but Node basically handles downloading packages to our project. These are the tools that we will need through the project. It sits in the background like a helper. https://nodejs.org/en/download

Also

1. If you're a Mac user, download Homebrew here: https://brew.sh/ - this is a package manager specifically for Mac. It will enable you to download specific dev-related software easily on the day if we need it.

2. Ensure you have Chrome web browser (the best for developers). Brave is also ok. Safari is fussier to use in my experience. If you're an Edge diehard then that should be fine, but Chrome works really well and it'll be the one I use. (And please do ensure you bring a laptop not just an iPad! Mac or PC makes no difference.)
